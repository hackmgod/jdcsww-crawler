# 📦 汽车公告全量数据爬虫 - 生产版

## 🎯 项目目标

爬取司机宝宝网 (jdcsww.com) 所有历史批次的新能源汽车公告数据，用于数据库导入。

### ✅ 必选字段（已实现）

| 字段 | 说明 | 示例 |
|------|------|------|
| 公告型号 | 车辆公告型号 | SQR5035XXYBEVH31 |
| 批次 | 公告批次号 | 403 |
| 燃料类型 | 车辆燃料类型 | 纯电动 |
| 生产企业 | 生产厂商名称 | 奇瑞商用车(安徽)有限公司 |

### 📊 预估数据量

- **批次范围**: 1-403 (截至2026年3月)
- **每批次约**: 30辆车
- **预估总量**: 约 12,000 辆车

```bash

```

## ✅ 混合动力车辆支持

### 已验证 ✅

**纯电动**: `rylx='C'` → 每批约30辆 ✅
**混合动力**: `rylx='O'` → 主要在早期批次 ✅

### 数据分布

| 燃料类型 | 批次分布 | 每批数量 | 预估总量 |
|---------|---------|---------|----------|
| 纯电动 | 1-403 | ~30辆 | ~12,000辆 |
| 混合动力 | 主要1-350 | 0-5辆 | ~500-1,000辆 |

### 代码实现

在 `crawl_production.py` 中已配置:

```python
fuel_types = [
    {'code': 'C', 'name': '纯电动'},
    {'code': 'O', 'name': '混合动力'},
]
```

### 验证结果

- ✅ 批次350找到混合动力车辆
- ✅ 参数 rylx=O 验证通过
- ✅ 同时爬取两种燃料类型

## 🛠️ 故障排除

### 问题1: 被拦截

**现象**: 显示"非正常访问"或"爬取数据嫌疑"

**解决**:
- 自动重试机制已内置
- 如持续失败，增加延迟时间
- 检查网络连接

### 问题2: 超时

**现象**: 请求超时

**解决**:
- 已有自动重试(最多3次)
- 检查网络稳定性
- 可以减少并发

### 问题3: Excel打开慢

**现象**: Excel文件很大，打开缓慢

**解决**:
- 正常现象，12,000行数据
- 可以分批处理
- 使用CSV格式替代
