# 增强版爬虫使用说明

## 📋 功能概述

增强版爬虫 `crawl_production_enhanced.py` 可以爬取**83个完整字段**，包括：

### 1. 生产企业信息（23个字段）
- 制造国、车辆名称、公告批次、产品号
- 中文品牌、公告型号、企业名称
- 生产地址、注册地址、通讯地址
- 电话、邮编、电子邮箱、传真、联系人
- 车辆类型、牌照类型、发布日期、目录序号
- 英文品牌、车型、产品ID号、法人代表

### 2. 车辆燃料参数（7个字段）
- 车辆燃料种类、车辆耗油、排放依据标准
- 排放标准提示、机动车环保网链接
- 是否新能源汽车、新能源汽车种类

### 3. 车辆参数（25个字段）
- 车辆总质量、额定载质量、车辆整备质量
- 车辆轴荷、车辆轴距、前/后轮距
- 钢板弹簧片数、半挂车鞍座最大载质量
- 接近角/离去角、车辆前悬/后悬
- 车辆轮胎规格型号、外形尺寸(长×宽×高)
- 货箱栏板尺寸、车辆识别代码(VIN)
- 车辆转向形式、车辆总轴数、最高车速
- 额定载客、前/后排允许载客
- 车辆传动形式、车辆轮胎数
- 准拖挂车总质量、载质量利用系数

### 4. 三电参数（28个字段）
**电芯信息（9个）**：
- 动力类型、电芯类型、电芯形状
- 电芯电压、电芯电量、电芯生产企业
- 续驶里程、电芯型号、电芯尺寸

**电池信息（7个）**：
- 电池容量、电池电量、电池管理系统BMS
- 电池电压、电池质量、电池生产企业
- 电池组合方式

**电机信息（6个）**：
- 电机型号、电机冷却方式
- 电机峰值功率/转速/转矩
- 电机生产企业、电机类型、电机数量

**电控系统（6个）**：
- 电控供应商、充电机生产企业
- 车载DC/DC变换器、充电机型号
- 充电机功率、高压配电盒(PDU)

## 🚀 使用方法

### 1. 运行爬虫

```bash
cd tram-inspection/jdcsww-crawler
python3 crawl_production_enhanced.py
```

### 2. 选择模式

```
选项:
1. 完整模式 - 爬取所有批次+详情页 (83个字段，非常慢)
2. 快速模式 - 仅爬取列表页 (6个字段，快速)
3. 测试模式 - 爬取单个批次测试 (385批次)
4. 继续上次中断的任务
5. 仅导出已有数据为Excel
```

## ⚡ 核心特性

### 1. 断点续传
- 自动保存进度到 `crawler_state.json`
- 中断后可从上次位置继续
- 支持用户中断（Ctrl+C）

### 2. 增量保存
- 每完成一个批次自动保存
- 数据实时写入 `all_vehicles_complete.json`
- 避免数据丢失

### 3. 防拦截策略
- 随机延迟（5-10秒）
- 详情页延迟（3-6秒）
- 每50个任务休息30秒
- 自动重试（最多3次）

### 4. 图片下载
- 自动下载车辆图片到 `images/` 目录
- 文件命名：`{公告编号}_{车名}_{序号}.jpg`
- 自动去重，避免重复下载

### 5. Excel导出
- 自动排序字段（重要字段在前）
- 冻结首行
- 格式化样式（蓝色表头）
- 支持大量字段（83+）

## 🎯 推荐使用流程

### 第一次运行（测试）
```bash
# 1. 先测试单个批次
python3 crawl_production_enhanced.py
# 选择: 3 (测试模式)

# 2. 检查数据质量
open data/production_enhanced/vehicles_complete_all.xlsx
```

### 正式运行（快速）
```bash
# 如果只需要基本信息（6字段）
python3 crawl_production_enhanced.py
# 选择: 2 (快速模式)
```

### 正式运行（完整）
```bash
# 如果需要完整数据（83字段）
python3 crawl_production_enhanced.py
# 选择: 1 (完整模式)
# 注意: 这将需要数天时间！
```

### 中断后继续
```bash
# 直接选择继续即可
python3 crawl_production_enhanced.py
# 选择: 4 (继续上次任务)
```

## 📈 性能估算

假设爬取403个批次，约8000辆车辆：

| 模式 | 详情页访问 | 延迟 | 预计时间 |
|------|-----------|------|---------|
| 快速模式 | 0 | 7秒/批次 | ~1小时 |
| 完整模式 | 8000次 | 5秒/详情页 | ~3-5天 |

**建议**: 先用快速模式获取所有车辆列表，再根据需要选择性地爬取详情页。

## 🔧 故障排除

### 问题1: 被拦截
```
解决: 增加延迟时间，修改代码中的 delay 参数
```

### 问题2: 图片下载失败
```
解决: 检查网络连接，部分图片可能不存在
```

### 问题3: 内存不足
```
解决: 分批爬取，使用 start_batch 和 end_batch 参数
```
