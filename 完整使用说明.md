## 🚀 快速开始

### 系统要求

- **Python版本**: Python 3.6+
- **操作系统**: Windows / macOS / Linux
- **依赖库**: 
  - `urllib` (Python内置)
  - `json` (Python内置)
  - `re` (Python内置)
  - `time` (Python内置)
  - `logging` (Python内置)
  - `openpyxl` (Excel导出，需安装)

### 安装依赖

```bash
# 安装 openpyxl（用于Excel导出）
pip install openpyxl

# 或使用系统包管理器
pip3 install openpyxl --break-system-packages
```

### 快速启动

```bash
# 1. 进入爬虫目录
cd jdcsww-crawler

# 2. 运行爬虫
python3 crawl_production_enhanced.py

# 3. 选择运行模式（见下文详细说明）
```

## 📚 详细功能说明

### 核心功能

#### 1. 数据爬取
- ✅ **列表页爬取**: 获取车辆基本信息（6个字段）
- ✅ **详情页爬取**: 获取完整83个字段数据
- ✅ **三电参数**: 包括电芯、电池、电机、电控系统
- ✅ **图片下载**: 自动下载车辆图片

#### 2. 数据管理
- ✅ **断点续传**: 中断后可以继续爬取
- ✅ **增量保存**: 每个批次完成后立即保存
- ✅ **状态跟踪**: 记录已完成和失败的批次
- ✅ **数据合并**: 自动合并新旧数据

#### 3. 数据导出
- ✅ **JSON格式**: 原始数据保存
- ✅ **Excel格式**: 格式化的Excel表格
- ✅ **图片保存**: 按公告编号分类保存

#### 4. 安全机制
- ✅ **预热机制**: 访问首页建立会话
- ✅ **完整请求头**: 模拟Chrome浏览器
- ✅ **随机延迟**: 10-20秒/15-30秒
- ✅ **智能休息**: 每20批次休息60秒
- ✅ **封禁检测**: 自动检测并停止

### 模式1：完整模式 ⭐⭐⭐⭐⭐

**用途**: 爬取所有批次的完整数据（包括详情页）

**特点**:
- 爬取批次范围: 1-403
- 爬取详情页: ✅ 是
- 数据字段: 83个
- 预计时间: 7-10天

**数据字段包括**:
```
基本信息:
- 批次、燃料类型、车辆名称、公告编号、公告型号
- 生产企业、发布日期、车型、等

三电参数:
- 电芯信息: 类型、型号、形状、电压、容量、生产企业
- 电池信息: 容量、电压、电量、质量
- 电机信息: 型号、功率、生产企业
- 电控系统: 型号、生产企业

车辆参数:
- 外形尺寸、质量参数、载客人数、最高车速
- 轮胎规格、轴数、接近角/离去角
```

**注意事项**:
1. **首次运行**: 需要7-10天完成
2. **网络稳定**: 需要稳定的网络连接
3. **不中断**: 建议在稳定环境下运行
4. **数据验证**: 每个批次完成后会立即保存

**适用场景**:
- ✅ 需要完整数据
- ✅ 长期项目
- ✅ 研究用途
- ❌ 紧急需求

## ⚙️ 配置参数说明

### 延迟配置（在 `__init__` 方法中）

```python
# 详情页之间延迟（秒）
self.detail_delay_min = 10.0   # 最小10秒
self.detail_delay_max = 20.0   # 最大20秒

# 批次间延迟（秒）
self.batch_delay_min = 15.0    # 最小15秒
self.batch_delay_max = 30.0    # 最大30秒

# 休息配置
self.rest_interval = 20        # 每20个批次
self.rest_duration = 60        # 休息60秒
```

### 自定义延迟参数

如果你想修改延迟，编辑 `crawl_production_enhanced.py` 第35-39行：

**保守策略**（更安全，更慢）:
```python
self.detail_delay_min = 15.0
self.detail_delay_max = 30.0
self.batch_delay_min = 20.0
self.batch_delay_max = 40.0
```

**平衡策略**（推荐）:
```python
self.detail_delay_min = 10.0
self.detail_delay_max = 20.0
self.batch_delay_min = 15.0
self.batch_delay_max = 30.0
```

**激进策略**（更快，风险更高）:
```python
self.detail_delay_min = 5.0
self.detail_delay_max = 10.0
self.batch_delay_min = 8.0
self.batch_delay_max = 15.0
```

## ❓ 常见问题解答

### Q1: 如何确认爬虫正常工作？

**A**: 运行测试模式：
```bash
python3 crawl_production_enhanced.py
# 选择 3 - 测试模式
```

预期结果：
- ✅ 成功爬取30辆车
- ✅ 生成Excel文件
- ✅ 无封禁提示

---

### Q2: 爬取中断了怎么办？

**A**: 使用模式4继续：
```bash
python3 crawl_production_enhanced.py
# 选择 4 - 继续上次中断的任务
```

**说明**:
- 自动跳过已完成批次
- 从中断点继续
- 数据不会丢失

---

### Q3: 如何只爬取特定批次？

**A**: 修改代码中的批次范围：

编辑 `crawl_production_enhanced.py` 第779行：
```python
# 原代码
crawler.run_full_crawl(start_batch=1, end_batch=403, crawl_detail=True)

# 修改为（例如只爬取380-390批次）
crawler.run_full_crawl(start_batch=380, end_batch=390, crawl_detail=True)
```

---

### Q4: 提示"非正常访问"怎么办？

**A**: 这说明触发了反爬虫机制

**解决方案**:
1. **立即停止**程序
2. **等待30-60分钟**
3. **检查IP状态**:
   ```bash
   python3 check_ip_status.py
   ```
4. **使用更保守的延迟参数**
5. **或更换网络**（手机热点）

---

### Q5: 如何查看爬取进度？

**A**: 查看日志文件：
```bash
tail -f data/production_enhanced/crawler_*.log
```

或查看状态文件：
```bash
cat data/production_enhanced/crawler_state.json
```

---

### Q6: 数据保存在哪里？

**A**: 数据保存在：
```
data/production_enhanced/
├── all_vehicles_complete.json    # JSON格式
├── vehicles_complete_all.xlsx    # Excel格式
└── images/                       # 图片文件夹
```

---

### Q7: 可以同时运行多个实例吗？

**A**: ❌ **不建议！**

**原因**:
- 可能相互干扰
- 增加被封禁风险
- 可能产生数据冲突

---

### Q8: 如何备份已爬取的数据？

**A**: 备份整个 `data` 文件夹：
```bash
# 备份
cp -r data/production_enhanced data/production_enhanced_backup_$(date +%Y%m%d)

# 或压缩备份
tar -czf backup_$(date +%Y%m%d).tar.gz data/production_enhanced/
```
